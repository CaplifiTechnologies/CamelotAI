The problem

Tab-hopping is not a strategy.

Switching between chat UIs loses context, duplicates spend, and makes it hard to see who contributed what. Camelot gives every model a named seat, one shared transcript, and rules that route mechanical work to free local inference.

Capabilities

Built for deliberation, not distraction

🪑

Named seats

Claude, Grok, local Ollama models, and custom agents — role-based routing, @mentions, and PASS when a seat has nothing to add.

🛡️

Cost Guard

Warn at ~10k tokens, hard-block at ~50k on paid seats. Vote tallies and aggregation always run on a free local model.

🌿

Local Only

Toggle with ⌘L — route through local models only. Paid seats go dark; API keys become optional.

🗳️

Votes + tasks

Open votes with confidence ballots, assign tasks to seats, and keep tally narration off your API bill.

🌿

Side threads

Branch from any message, explore in isolation, merge back as a summary — export includes full branches.

📦

Desktop-native

Electron + Next.js, SQLite persistence, setup wizard, secure credential storage, and a macOS .dmg for arm64.

"Local models lead. Paid APIs are backup. You keep the last word."

— Camelot routing principle

Development history

Version stack

Every release archived. Newest first.

v0.2.2 · Local-First

Current 2026-06-21

Local agents lead by default. Paid API seats are opt-in.

  • Local agent seat — full agent loop via HTTP, not raw chat completion
  • Priority routing — local → fallback local → paid seats (when enabled)
  • Setup wizard — guided onboarding for local models and optional API keys
  • Cost visibility — per-message usage estimates and expensive-request confirmation
  • Offline model vault — hot-tier defaults when a local model library is mounted
  • Seat toggles — enable or disable paid seats without restarting

v0.2.1 · Packaged

Shipped 2026-06-20

macOS .dmg installable from Applications.

  • Embedded Next server — production smoke tests for packaged builds
  • UserData database — SQLite in application data, not inside the app bundle
  • Filesystem bridge hardening — path checks block symlink escape
  • Production Electron fixes — resolves bundle path issues on install

v0.2.0 · Round Table

Shipped 2026-06-20

Production packaging and boardroom UX improvements.

  • Side-thread merge — summaries post as a dedicated seat
  • Export — full side-thread sections in markdown export
  • PASS normalization — consistent skip semantics across seats
  • Exchange limit — caps seat-to-seat replies between your messages
  • Filesystem bridge — scoped read/write/list in the desktop app
  • Auth clarity — clearer token expiry and re-login messaging

v0.1.0 · Foundation

Archived 2026-06-19

First feature-complete desktop build.

  • Boardroom chat — user-driven turns, SQLite persistence
  • Seats — Claude, Grok, local Ollama; health check per provider
  • Votes + tasks — confidence ballots, local-only tally, task assignment
  • Side threads — branch, isolate, merge-back
  • Cost Guard + Local Only — spend thresholds and ⌘L routing
  • Invite modal — seat any running Ollama model

Get Camelot

Run it on your Mac.

Desktop app for macOS arm64. Requires Node 20+ and Ollama for free local seats. Paid API seats are optional. 

git clone https://github.com/CaplifiTechnologies/CamelotAI.git
cd CamelotAI
npm install
npm run prisma:generate
npm run prisma:push
npm run desktop

Or install the .dmg when a release is published. Stack: Electron 30 · Next.js 14 · Prisma · MIT License
